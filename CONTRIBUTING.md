# Contributing to NarratorEngine

This project is a modular Python framework for building terminal-based narrative games. Contributions of all kinds are welcome!

## Ways to Contribute

You can contribute in several ways:

* Reporting bugs
* Suggesting new features
* Improving documentation
* Refactoring code
* Adding examples or demos

## Getting Started

1. Fork the repository

2. Clone your fork:

   git clone https://github.com/TheDanskiSon09/NarratorEngine

3. Create a new branch:

   git checkout -b feature/your-feature-name

## Development Guidelines

### Code Style

* Use clear and descriptive names
* Follow Python conventions (PEP 8 where reasonable)
* Keep code simple and readable
* Avoid unnecessary complexity

### Project Structure

Please keep the architecture clean - do not mix engine logic!

---

### Dependencies

The core engine should remain:

* Lightweight
* Free of external dependencies

If your feature requires a third-party library, then:

* Place it in `requirements.txt` file
* Or make it optional

---

## Submitting Changes

1. Commit your changes with clear messages:

   feat: add audio backend
   fix: resolve input recursion bug
   refactor: simplify level loading

2. Push your branch:

   git push origin feature/your-feature-name

3. Open a Pull Request

---

## Pull Request Guidelines

Please ensure:

* Code is clean and readable
* No breaking changes without explanation
* Changes are tested manually
* PR description clearly explains what was added or changed

---

## Issues

When creating an issue, include:

* Clear description
* Steps to reproduce (if bug)
* Expected vs actual behavior

---

Thank you for contributing ❤️
